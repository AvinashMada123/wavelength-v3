Sequence Flow Builder — Design Summary

Review and confirm before we write the spec

Core Concept

Replace linear step-based sequences with a visual flow builder using React Flow. Drag-and-drop nodes on a canvas, connect with edges, branch on outcomes.

Node Types (v1)

Category Node Description
Action Voice Call Queue a call. Quick retry config (N times, X hrs apart) or wire retries in flow.
WhatsApp Template Send pre-approved template. Params can be AI-filled or manual.
WhatsApp Session Freeform message within 24hr window.
AI Generate + Send Generate content with AI prompt, send via WhatsApp. Two modes: fill template vars OR full message.
Control Condition Branch on: call outcome (picked up/no answer/busy/failed), post-call analysis (interest, goal, sentiment), reply content, lead fields.
Delay / Wait Wait X hours/days. Configurable send window (business hours) per node.
Wait for Event Wait for reply/call outcome with mandatory timeout (default 24hrs). Timeout branch continues flow.
Terminal Goal Met Milestone marker — doesn't end flow, marks achievement. Multiple goals per flow.
End Flow complete for this lead.

Phase 2: Generate Document node (AI-filled PDF via HTML template, sent as WhatsApp attachment)

Triggers (v1)

Keep existing: post_call, manual, campaign_complete. Architected for future triggers.

Call Outcome Branching

Detailed outcomes feed back into flow:

  • Connection: Picked up / No answer / Busy / Failed
  • Post-call analysis: Interest level, goal outcome, sentiment, callback requested

Retries: Quick config on call node for simple cases. Explicit flow wiring for complex strategies (e.g., WhatsApp between retry attempts).

Business Hours

Configurable time windows per node (calls & WhatsApp). Org-level defaults with per-node override. Option to disable and send anytime.

Draft / Publish Model

  • Edit freely in draft mode
  • Publish makes it live — all leads migrate to new version
  • Deleting a node with active leads: system warns and requires removing leads first

Simulation & Testing

  • Visual dry-run: Simulate on canvas, pick outcomes at each branch, watch flow highlight path
  • Live test: Enroll yourself with test flag, real messages with compressed delays
  • Journey replay: Click any lead on canvas to see their actual path highlighted

Lead Integration

  • Add/remove leads from flows manually at any time
  • Lead profile: New tab with full flow history timeline
  • Canvas view: Panel showing leads in flow, click to highlight their path
  • Full traceability for debugging

Analytics

  • Canvas badges: Per-node stats (passed/dropped/success rate)
  • Analytics page: Conversion rates, drop-off, time between nodes, goal completion

Multi-flow

Flows are independent. max_active_per_lead prevents over-contact.
