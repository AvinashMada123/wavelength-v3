Sequence Flow Builder — Revised Design (Post-Audit)

Updated with all audit findings. Changes from v1 highlighted in green.

1. Core Architecture

Replace linear step-based sequences with a visual flow builder using React Flow. Drag-and-drop nodes on a canvas, connect with edges, branch on outcomes.

NEW — Graph Storage: Replace step_order integer with proper graph model: FlowNode (id, type, position, config) + FlowEdge (source_node, target_node, condition_label). DAG structure stored as nodes + edges, not linear steps.

NEW — Graph Traversal Engine: New engine that walks the DAG. On node completion, evaluates outgoing edges, picks the matching branch, activates the next node. Replaces linear step iteration.

2. Node Types (v1)

Category Node Description
Action Voice Call Queue a call. Quick retry config (N times, X hrs apart) or wire retries in flow. Bot config retries ignored inside flows.
WhatsApp Template Send pre-approved template. Params: manual or AI-filled.
WhatsApp Session Freeform message within 24hr window. NEW: If window expired → auto-fallback to WhatsApp Template node (configurable) or skip with "expired" branch output.
AI Generate + Send Two modes: fill template vars OR generate full message. Sent via WhatsApp.
Control Condition Branch on: call outcome, post-call analysis, reply content, lead fields.
Delay / Wait Wait X hours/days. Configurable send window per node.
Wait for Event Wait for reply/call outcome. Mandatory timeout (default 24hrs). Timeout branch.
Terminal Goal Met Milestone marker — doesn't end flow. Multiple per flow.
End Flow complete for this lead.

Phase 2: Generate Document node (AI-filled PDF via HTML template)

3. Critical Infrastructure (Pre-requisites)

These must be built BEFORE the flow builder:

Call Outcome Feedback Loop Wire plivo_event() to update sequence touchpoints on call end. Map Plivo statuses to: picked_up, no_answer, busy, failed, voicemail. Preserve granularity — don't collapse statuses.
Plivo Status Granularity Fix current mapping that collapses busy/timeout/no-answer into one value. Store raw Plivo status + normalized enum.
Node Transition Audit Trail New FlowTransition table: instance_id, from_node, to_node, timestamp, outcome_data. Powers journey replay and debugging.
Business Hours System Org-level default hours (e.g., Mon-Sat 9AM-7PM IST). Per-node override. "Send anytime" toggle.

4. Draft / Publish Model (REVISED)

Changed from v1: All leads no longer auto-migrate on publish.

  • Edit freely in draft mode
  • Publish creates a new version — existing leads continue on their version, new enrollments get the new version
  • Manual "re-enroll" option to move specific leads to new version
  • Deleting a node with active leads: system warns, requires removing leads first
  • Version history visible — can see what changed between versions

5. Flow Validation (NEW)

Before publishing, validate:

  • Every path reaches an End node (no dangling branches)
  • No disconnected nodes
  • No infinite loops without a delay/wait (prevents instant cycling)
  • All condition nodes have at least 2 outgoing branches
  • All action nodes have required config filled in
  • Warnings (non-blocking): unused branches, very short delays

6. Rate Limiting & Flood Protection (NEW)

  • Per-lead daily cap: Max messages/calls per lead per day (org-configurable, default: 5)
  • Per-lead hourly cap: Max 2 messages per hour per lead
  • Cooldown: Min 60s between any two actions to same lead (existing)
  • When cap hit → action queued for next available window, not dropped

7. UX Guardrails & Onboarding (NEW)

Target users are Indian coaches — not technical. Canvas must be approachable.

  • Auto-layout mode: Default to structured top-down layout. Nodes snap to grid, edges auto-route. Free-form positioning as opt-in.
  • Flow templates: Pre-built starter flows for common scenarios:
    • "Post-call follow-up" — call → condition → WhatsApp or retry
    • "No-answer recovery" — call → no answer → WhatsApp → delay → retry call
    • "Interested lead nurture" — condition on interest → personalized AI messages
  • Undo/Redo: Full undo/redo stack on canvas (Ctrl+Z/Ctrl+Y)
  • Guided first-time experience: Tooltip tour on first visit
  • Node palette: Sidebar with drag-and-drop nodes, grouped by category with clear labels

8. Simulation & Testing

  • Visual dry-run: Simulate on canvas, pick outcomes at each branch, watch path highlight. No messages sent.
  • Live test: Enroll yourself with test flag, real messages, compressed delays (minutes instead of hours).
  • Journey replay: Click any real lead on canvas to see their actual path highlighted with outcomes and timestamps at each node.

9. Lead Integration

  • Add/remove leads from flows manually at any time
  • Lead profile: New tab with full flow history timeline (which flows, which nodes, what happened)
  • Canvas view: Panel showing leads in this flow, click to see their path
  • NEW — Transition log: Every node transition recorded with timestamp and outcome data. Full audit trail for debugging.

10. Analytics

  • Canvas badges: Per-node stats (passed/dropped/success rate)
  • Analytics page: Conversion rates, drop-off per node, time between nodes, goal completion rates

11. Triggers & Multi-flow

  • Triggers: Keep existing (post_call, manual, campaign_complete). Extensible architecture for future triggers.
  • Multi-flow: Independent flows. max_active_per_lead prevents over-contact.
  • Business hours: Org-level defaults + per-node override + "send anytime" toggle.

12. Migration Strategy

Existing linear sequences must continue working:

  • Auto-convert existing templates to single-path flows (step 1 → step 2 → ... → End)
  • Active instances complete on the linear engine — no mid-flight migration
  • New enrollments use the flow engine
  • Deprecate linear engine after all active instances complete

13. What We Keep

  • anthropic_client.py — AI generation, fully reusable
  • messaging_client.py — WhatsApp sending, fully reusable
  • QueuedCall system — voice call queuing, extra_vars already carries touchpoint ID
  • Trigger evaluation logic from sequence_engine.py
  • Reply handling + WhatsApp session window tracking
  • Instance/Touchpoint models — extended with new columns, not replaced
